What Are HTTP Security Headers?

        
HTTP security headers are directives sent by a web server that tell browsers how to behave when handling a site's content. They form a critical defense layer against cross-site scripting (XSS), clickjacking, protocol downgrade attacks, and data injection.

        

          
CSP
Content-Security-Policy
+10 to -25
The single most effective header against XSS. It specifies which sources the browser should accept for scripts, styles, images, and other resources.

          
HSTS
Strict-Transport-Security
+5 to -20
Forces browsers to use HTTPS for all future requests. Prevents SSL-stripping attacks.

          
XFO
X-Frame-Options
0 to -20
Prevents pages from being loaded in iframes. Primary defense against clickjacking.

          
RP
Referrer-Policy
+5 to 0
Controls how much referrer information is included with requests.

          
XCTO
X-Content-Type-Options
0 to -5
Prevents browsers from MIME-sniffing a response away from the declared Content-Type.

          
CORS
Cross-Origin Resource Sharing
0 to -50
Controls which external origins can access your site's resources.

          
RED
Redirection (HTTP to HTTPS)
0 to -20
Tests whether HTTP requests are properly redirected to HTTPS.

          
SRI
Subresource Integrity
+5 to -50
Ensures third-party scripts and stylesheets haven't been tampered with.

        


        

          
Scoring Methodology

          
Scoring follows the Mozilla Observatory algorithm v5. The base score starts at 100 and is adjusted by penalties and bonuses across 11 security tests. Extra credit only counts if your base score is 90 or above.

Is this security headers checker free?
Yes, completely free with no signup required. You can check as many websites as you want.

        
How does the checker work?
The checker sends up to 3 requests to the target site: an HTTPS GET to read response headers, an HTTP request to test redirect behavior, and a CORS probe. It analyzes results against 11 security tests based on the Mozilla Observatory algorithm v5.

        
Will scanning cause any damage?
No. The checker sends up to 3 lightweight HTTP requests — identical to what a normal browser does when visiting a page. Completely non-intrusive.

        
What is a good grade?
Grades range from A+ to F across 13 levels. An A+ requires a score of 100 or above. Anything below C (under 55) indicates significant gaps.

        
How does scoring compare to Mozilla Observatory?
Our scoring uses the Observatory algorithm v5 with additional checks like SRI, CORS, and cookie security analysis. The 13-grade scale matches Observatory.
